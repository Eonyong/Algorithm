## 최소 신장 트리(MST)

## 특징

~~~
최소 가중치 스패닝 트리라고도 불린다.
최소 연결 트리로써 n개의 노드를 가지고 있으면 (n - 1)개의 간선을 으로 연결되어 필연적으로 트리 형태를 이룬다.
하나의 그래프에는 다양한 신장 트리가 존재할 수 있다.
	-> 그 중 가중치 합이 가장 작은 것이 최소 신장 트리
모든 정점들이 서로 연결되어 있다.
사이클을 포함해서는 안된다.
DFS, BFS를 통해 그래프에서 신장 트리를 찾을 수 있다.
완전 그래프에선 n개의 노드일 때 신장 트리의 개수는 n^(n - 2)개가 존재
~~~



## MST 구현 방법

~~~
Prim's algorithm
Kruskal's algorithm
Reverse-delete algorithm
Borůvka's algorithm
~~~



### Prim's Algorithm

~~~
가중치가 있는 연결된 무향 그래프의 모든 꼭짓점을 포함한 최소 비용 생성나무를 찾는 알고리즘이다.
우선순위 큐를 필요로 하는 노드 위주의 알고리즘이다

시간 복잡도(노드 수 V, 간선 수 E)
	이진 힙을 이용했을 때를 기준으로 O(ElogV)의 시간복잡도를 가진다.
	피보나치 힘을 이용한 경우에는 O(E + VlogV)까지 떨어진다.
	
작동 순서는
	1. 그래프에서 하나의 꼭짓점을 선택하여 트리를 만든다.
	2. 그래프의 모든 변이 들어 있는 집합을 만든다.
	3. 모든 꼭짓점이 트리에 포함되어 있지 않는 동안 트리와 연결된 변 가운데 트리 속의 두 꼭짓점을 연결하지 않는 		가장 가중치가 작은 변을 트리에 추가한다.
~~~



### Kruskal's Algorithm

~~~
최소 비용 신장 부분 트리를 찾는 알고리즘이다.
정렬과 서로소 집합 자료구조를 필요로 하는 간선 위주의 알고리즘이다.

작동 순서는
	1. 그래프 간선을 가중치의 오름차순으로 정렬한다.
	2. 정렬된 간선 리스트 중에서 사이클이 형성되지 않는 간선들을 선택한다.
		1. 그 중 가장 낮은 가중치를 가진 간선을 선택한다.
		2. 사이클을 형성 여부를 판단하여 트리 집합에 추가한다.
	3. 2번의 과정을 반복하여 집합의 수가 (노드의 수 - 1)개가 되면 중지한다.
		
다른 방식으로는 그래프에서 변을 제거하는 방식이다. 해당 방식의 순서는
	1. 그래프에서 꼭짓점 또는 트리를 분히래 내지 않는 최대 가중치의 변을 제거한다.
	2. 그래프에 N - 1개의 변만 남을 때까지 위 과정을 반족한다.
	3. 그래브에 N - 1개의 번이 남으면 최소 비용 생성 트리이다.
	
code link: https://youbin-shin.github.io/posts/cs-algorithm-3/
~~~



### Borůvka's algorithm

~~~
작동 순서는
	1. 모든 노드에서 가장 작은 가중치를 가지는 간선을 설택한다.
	2. 선택된 간선 양쪽의 정점을 하나로 만든다.
	3. 중복 간선이 생기는 경우 가장 작은 것 하나만 선택한다.
	4. 위 과정을 모든 정점이 연결될 때까지 반복한다.
~~~
